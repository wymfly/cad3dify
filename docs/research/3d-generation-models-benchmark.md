## 二、技术架构深度对比

### 2.1 3D 表示方式

这是三个模型最本质的差异，决定了它们在不同场景下的表现。

**Hunyuan3D-2.1 — 隐式场 → Mesh + UV 展开 + 独立纹理管道**

生成分两步：(1) 用 3.3B MoE DiT 从图片推断 3D 隐式场，提取 mesh 后固定 remesh 到 40K 面；(2) 用独立的纹理管道（光栅化器 → 纹理修复 → RealESRGAN 超分）在 UV 空间上绘制 baseColor 和 metallicRoughness 贴图。两步之间解耦——几何和纹理由不同模型分别生成。

- 优势：纹理管道可单独优化，PBR 材质分离度高（金属度/粗糙度独立通道）
- 劣势：接缝和遮挡区域纹理容易出现瑕疵（"涂抹感"），纹理与几何的一致性依赖修复算法
- 面数固定 40K，无法调节

**TripoSG — SDF → Marching Cubes**

直接预测高分辨率 Signed Distance Field (SDF)，通过 Marching Cubes 提取等值面。不做任何降面或纹理处理，输出原始密度的三角网格。

- 优势：几何细节保留最完整，SDF 天然生成闭合流形（watertight mesh），面数密度高
- 劣势：无纹理输出，面数不可控（60万–210万），文件体积大
- 本质上是一个"几何引擎"

**TRELLIS.2 — Structured Latent (SLat) → O-Voxel 解码**

用 DINOv3 视觉特征 + 4B DiT 模型在结构化隐空间 (SLat) 中同时编码几何和外观，然后通过 O-Voxel 解码器一体化输出 mesh + 纹理。几何和纹理从同一个 latent 解码，而非分步生成。

- 优势：纹理与几何一致性好（同源解码），面数可配（simplify 参数），显存效率高（low_vram 模式）
- 劣势：SLat 表示的几何细节上限不如 SDF，极小特征可能丢失
- 面数默认 100K，可通过 simplify 参数调节

### 2.2 纹理生成架构差异

| 维度 | Hunyuan3D-2.1 | TripoSG | TRELLIS.2 |
|------|-------------|---------|-----------|
| 纹理生成方式 | 独立管道（后处理贴图） | 无 | 一体化解码（SLat → 纹理） |
| PBR 通道 | baseColor + metallicRoughness | 无 | baseColor + PBR（一体） |
| 纹理分辨率 | 高（RealESRGAN 4x 超分） | 无 | 中高（O-Voxel 投影） |
| 接缝控制 | 需修复（mesh_inpaint） | 无 | 自然过渡（同源生成） |
| 遮挡区域 | 容易出现涂抹/模糊 | 无 | 相对自然（SLat 推断） |
| 金属/粗糙度分离 | 独立通道，控制精确 | 无 | 混合通道 |

**关键差异**：Hunyuan3D-2.1 的纹理更"精致"但有接缝风险；TRELLIS.2 的纹理更"自然"但分辨率略低。在需要严格 PBR 材质控制（如区分金属/非金属区域）的场景下，Hunyuan3D-2.1 更优。在追求整体视觉自然感的场景下，TRELLIS.2 更优。

### 2.3 几何精度对比

| 维度     | Hunyuan3D-2.1 | TripoSG  | TRELLIS.2 |
| ------ | :-----------: | :------: | :-------: |
| 输出面数   |    40K（固定）    | 60万–210万 | ~100K（可调） |
| 小细节保留  |      ★★★      |  ★★★★★   |    ★★★    |
| 锐边/硬表面 |      ★★★      |   ★★★★   |   ★★★★    |
| 曲面连续性  |     ★★★★      |  ★★★★★   |   ★★★★    |
| 全局拓扑   |      ★★★      |   ★★★★   |   ★★★★    |
| 薄壁结构   |      ★★       |   ★★★    |    ★★     |
| 对称性    |      ★★       |   ★★★    |    ★★★    |

**关键差异**：TripoSG 的 SDF → Marching Cubes 路线在几何精度上有本质优势。百万级面数意味着锐边不会被磨圆、小孔洞不会消失、细长结构不会断裂。Hunyuan3D-2.1 的 40K 固定 remesh 是它最大的几何瓶颈——remesh 过程中不可避免地丢失细节。TRELLIS.2 的 100K 面数是折中方案。

## 四、真实应用场景分析

### 4.1 场景覆盖矩阵

从 3D 建模工程师的实际工作流出发，逐场景评估。

#### 工业设计 / 概念验证

| 具体需求 | 最佳选择 | 原因 |
|---------|:---:|------|
| 设计师快速验证造型比例（出 5-10 个方案筛选） | **TripoSG** | 14s/个，5 个方案仅需 70s；白模足够评估比例和造型语言 |
| 概念设计评审会的 3D 可视化 | **TRELLIS.2** | 带纹理，59s 出图，可直接嵌入演示文稿或 Web 3D viewer |
| 客户提案的产品渲染 | **Hunyuan3D-2.1** | PBR 材质渲染效果最佳，放入 Three.js/Unreal 中展示最有说服力 |
| CMF（色彩/材质/工艺）探索 | **Hunyuan3D-2.1** | PBR 金属度/粗糙度独立控制，可在渲染器中切换材质方案 |

#### 3D 打印

| 具体需求 | 最佳选择 | 原因 |
|---------|:---:|------|
| FDM/SLA 打印（对面数要求适中） | **TRELLIS.2** | 100K 面数适合切片，带纹理可用于全彩打印（MJF/Polyjet） |
| 手办/艺术品高精度打印 | **TripoSG** | 百万级面数保留所有细节，导入切片软件后无需补面 |
| 打印前造型预览 | **TripoSG** | 白模即可判断是否适合打印（壁厚、悬垂、支撑需求） |
| 全彩 3D 打印（Polyjet/MJF） | **TRELLIS.2** | 纹理直接映射到打印色彩，无需额外上色 |

#### 数字内容 / Web / AR

| 具体需求 | 最佳选择 | 原因 |
|---------|:---:|------|
| 电商产品 3D 展示 | **TRELLIS.2** | GLB 直接兼容主流 3D viewer，纹理自然，文件大小适中 |
| WebXR / AR 体验 | **TRELLIS.2** | GLB 标准格式，100K 面数对移动端友好 |
| 游戏 LOD0 资产（中近景） | **Hunyuan3D-2.1** | 40K 面数接近游戏中景标准，PBR 材质直接用 |
| 游戏 LOD2+ 资产（远景占位） | **TripoSG** | 高精度几何可精确 decimation 到任意目标面数 |
| 影视预览/镜头 Layout | **TripoSG** | 高精度白模适合空间关系验证和构图预览 |

#### 逆向工程 / 几何参考

| 具体需求 | 最佳选择 | 原因 |
|---------|:---:|------|
| 照片→参考几何→CAD 重建 | **TripoSG** | 最高几何精度，作为 CAD 参数化建模的形状参考不可替代 |
| 导入 ZBrush/Mudbox 精雕 | **TripoSG** | 百万级面数可直接雕刻，无需 subdivision |
| 导入 Blender 做二次编辑 | **TRELLIS.2** | 100K 面数 + 纹理，编辑友好且视觉完整 |
| 导入 Fusion360/SolidWorks 做参考 | **TripoSG** | 高精度白模做参考体最准确 |

### 4.2 三个模型不可互相替代的领域

以下场景中，某个模型具有不可替代的优势：

| 不可替代场景 | 唯一选择 | 为什么其他模型不行 |
|------------|:---:|------------|
| 需要 60 万+ 面数的高精度几何 | **TripoSG** | Hunyuan3D 固定 40K，TRELLIS.2 上限约 100K，均无法达到 |
| 锐边零件的精确棱线还原 | **TripoSG** | 40K remesh 会磨圆所有锐边 |
| 逆向工程几何参考 | **TripoSG** | 几何精度是硬性要求 |
| PBR 金属度/粗糙度独立控制 | **Hunyuan3D-2.1** | TRELLIS.2 的 PBR 通道是一体解码，分离度不如独立管道 |
| 产品级高分辨率纹理渲染 | **Hunyuan3D-2.1** | RealESRGAN 4x 超分后纹理分辨率最高 |
| 2GB 显存限制下的部署 | **TRELLIS.2** | low_vram 模式空闲仅 2.3GB，其他模型做不到 |
| 面数可配的灵活输出 | **TRELLIS.2** | simplify 参数可从 1K 到 1M 任意配置 |

### 4.3 "只追求质量不考虑速度，Hunyuan3D-2.1 能否替代其他两个？"

**不能。** 具体原因：

| 对比维度 | Hunyuan3D-2.1 vs TripoSG | 结论 |
|---------|--------------------------|------|
| 几何细节 | 40K 面 vs 60万–210万面 | TripoSG 碾压 |
| 锐边保留 | remesh 磨圆 vs 原始锐利 | TripoSG 胜 |
| 小孔洞/凹槽 | 40K 面可能丢失 vs 完整保留 | TripoSG 胜 |
| 纹理 | 有 PBR vs 无 | Hunyuan3D 胜 |

| 对比维度 | Hunyuan3D-2.1 vs TRELLIS.2 | 结论 |
|---------|---------------------------|------|
| 纹理接缝 | 后贴图有接缝风险 vs 一体解码无接缝 | TRELLIS.2 胜 |
| 遮挡区纹理 | 容易涂抹/模糊 vs 自然推断 | TRELLIS.2 胜 |
| 面数灵活性 | 固定 40K vs 可配 1K–1M | TRELLIS.2 胜 |
| PBR 分离度 | 独立通道精确控制 vs 一体通道 | Hunyuan3D 胜 |
| 纹理分辨率 | RealESRGAN 超分，最高 vs 中高 | Hunyuan3D 胜 |

**总结**：Hunyuan3D-2.1 在 PBR 材质精度和纹理分辨率上领先，但在几何精度、纹理一致性、面数灵活性上不如其他两个模型。三者无法互相替代。

## 五、三者共同的能力边界

作为"图片→3D mesh"的 AI 模型，以下需求是三个模型**都无法满足**的：

| 需求类别 | 具体场景 | 原因 | 应使用的工具 |
|---------|---------|------|------------|
| 精密工程 | 公差配合零件（±0.01mm） | AI 生成无尺寸精度保证 | SolidWorks / Fusion360 |
| 参数化建模 | 可编辑特征树（修改圆角半径、孔径） | 输出是死网格，无特征历史 | CADPilot 精密管道 / CadQuery |
| 装配约束 | 多零件配合（同轴、相切、间隙配合） | 单体生成，无装配语义 | SolidWorks Assembly |
| 工程图纸还原 | 按标注尺寸精确复现三视图 | 图片输入无法提取精确尺寸 | CADPilot 精密管道 |
| 拓扑优化 | 按载荷/约束优化材料分布 | 无 FEA 分析能力 | Altair Inspire / nTopology |
| Class-A 曲面 | 汽车外覆盖件 G2/G3 连续曲面 | 三角面片非 NURBS | Alias / ICEM Surf |
| GD&T 标注 | 形位公差（平行度、同轴度） | 无工程语义 | CAD + CMM |
| DFM 分析 | 拔模角、壁厚、底切、熔接线 | 无制造工艺约束 | Moldflow / DFMPro |
| 薄壁/空心结构 | 均匀壁厚的壳体 | AI 生成实心体为主 | Shell 命令 / CAD 参数化 |
| 精确对称 | 严格左右/旋转对称 | AI 不保证数学对称 | Mirror/Pattern 命令 |

**这些场景正是 CADPilot 精密管道（LLM → CadQuery → STEP）的覆盖范围。**

## 六、管线集成建议

### 6.1 CADPilot 有机管道推荐分层策略

```
用户输入图片/文本描述
  │
  ├─ 快速预览模式 ────→ TripoSG (14s, 白模)
  │   └─ 场景：设计师快速筛选方案、客户初步确认造型方向
  │
  ├─ 标准模式 ────────→ TRELLIS.2 (59s, 带 PBR 纹理)
  │   └─ 场景：产品展示、Web/AR、3D 打印准备、日常使用
  │
  └─ 高质量模式 ──────→ Hunyuan3D-2.1 (170s, 最高品质 PBR)
      └─ 场景：产品渲染、CMF 探索、最终交付级输出
```

### 6.2 互补协作模式

| 协作模式 | 流程 | 总时间 | 典型场景 |
|---------|------|--------|---------|
| 快速筛选→精细出图 | TripoSG 预览 → 确认 → TRELLIS.2 出图 | ~75s | 日常使用，覆盖 80% 需求 |
| 白模→精雕 | TripoSG → 导入 ZBrush 精雕 + 手绘纹理 | 14s + 手动 | 手办/艺术品制作 |
| 双路对比择优 | TRELLIS.2 + Hunyuan3D 同时生成 → 选优 | ~170s | 重要项目 |
| AI 参考→参数化重建 | TripoSG 白模 → CADPilot 精密管道参数化 | 视复杂度 | 逆向工程辅助 |
| 几何+纹理混合 | TripoSG 几何 + TRELLIS.2 纹理风格迁移 | ~75s | 需要高精度几何 + 纹理 |

### 6.3 RTX 5090 32GB 共存部署方案

| 方案 | 常驻模型 | 空闲占用 | 推理可用 | 适用场景 |
|------|---------|---------|---------|---------|
| **A（推荐）** | TripoSG + TRELLIS.2 | 5.3 GB | 27.3 GB | 日常使用，覆盖 90% 需求 |
| B | Hunyuan3D + TRELLIS.2 | 18.6 GB | 14 GB | 偏重渲染质量 |
| C | 三者全加载 | ~21.6 GB | ~11 GB | 全功能但显存紧张 |

方案 A 最优：TripoSG (3GB) + TRELLIS.2 low_vram (2.3GB) 仅占 5.3GB，绝大多数场景够用。需要最高品质时按需启动 Hunyuan3D-2.1。
