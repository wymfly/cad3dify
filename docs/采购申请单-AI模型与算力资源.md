## 一、项目概述

### 1.1 项目背景

CADPilot 是一个 AI 驱动的工业级 3D CAD 模型生成平台，旨在将**自然语言描述、工程图纸、参考图片**等多模态输入，自动转化为可直接用于 3D 打印和 CNC 加工的工业级模型文件（STEP / STL / 3MF / G-code）。

传统的 CAD 建模需要工程师使用 SolidWorks、CATIA 等专业软件手动操作，学习门槛高、建模周期长。CADPilot 通过 AI 技术将这一过程自动化，**将数小时的手工建模缩短为分钟级的自动生成**，大幅降低 3D 设计与打印的技术门槛。

### 1.2 当前进展

项目已构建完成基于 **LangGraph** 的端到端管线架构，涵盖 19 个功能节点，支持三种输入模式（自然语言 / 工程图纸 / 有机形态）的自动路由与处理。前端采用 React + Three.js 实现 3D 可视化工作台，后端采用 FastAPI 提供 RESTful API 与 SSE 实时事件流。

**核心架构：**

```
用户输入 (自然语言 / 工程图纸 / 参考图片)
  ↓
┌─ 路由层 ─────────────────────────────────────────────┐
│  TEXT → 意图解析    DRAWING → 图纸分析    ORGANIC → 有机形态分析  │
└──────────────────────────────────────────────────────┘
  ↓
  用户确认（HITL 人机协作）
  ↓
┌─ 生成层 ─────────────────────────────────────────────┐
│  参数化模板 / LLM 代码生成 / Neural 3D 模型生成             │
└──────────────────────────────────────────────────────┘
  ↓
┌─ 后处理层 ────────────────────────────────────────────┐
│  网格修复 → 缩放对齐 → 布尔装配 → 切片出码 → 打印优化        │
└──────────────────────────────────────────────────────┘
  ↓
  输出：STEP / STL / 3MF / G-code / 参数化源码 / 打印建议报告
```

## 二、功能列表

### 2.1 多模态输入与智能分析

| 功能 | 说明 |
|------|------|
| **自然语言意图解析** | 用户输入口语化描述（如"直径50mm的法兰盘，4个螺栓孔"），AI 自动解析为结构化工程参数 |
| **工程图纸智能识别** | 通过多模态视觉大模型读取 PNG/JPG 工程图纸，自动提取零件类型、尺寸、特征 |
| **有机形态生成** | 支持文本描述 + 参考图片生成有机形状（如人体工学外壳、艺术造型等非规则几何体） |
| **参数确认（HITL）** | AI 分析后暂停，展示解析结果供用户确认和修正，确保参数准确 |

### 2.2 3D 模型生成

| 功能 | 说明 |
|------|------|
| **参数化模板引擎** | 内置 17 个参数化模板（螺栓、法兰、齿轮、支架、壳体等），参数代入即可精确生成 |
| **LLM 代码生成** | 当模板无法覆盖时，LLM 自动生成 CadQuery 参数化 CAD 代码并执行 |
| **智能修复（SmartRefiner）** | 三层防线验证：静态参数校验 → 包围盒校验 → 视觉对比，最多 3 轮自动修复 |
| **Neural 3D 生成** | 4 种 3D 生成模型策略（Hunyuan3D / Tripo3D / SPAR3D / TRELLIS），支持 SaaS + 本地双部署 |
| **建模策略匹配** | 规则引擎自动匹配 7 种零件类型（回转体、阶梯轴、板件、支架、壳体、齿轮、通用件）的最佳建模策略 |

### 2.3 后处理与打印优化

| 功能 | 说明 |
|------|------|
| **网格修复** | MeshLib 体素化修复 + Manifold3D 流形保证 + Neural 神经网络修复，三策略可选 |
| **网格缩放与对齐** | 均匀缩放 + Z 轴贴合 + XY 中心化，适配打印床 |
| **布尔装配** | Manifold3D 精确布尔运算（并/差/交），支持多零件组装 |
| **切片出码** | 集成 PrusaSlicer / OrcaSlicer，自动生成 G-code |
| **晶格填充** | TPMS 拓扑优化多孔结构（Gyroid/Diamond/Primitive），可调孔隙率 |
| **打印方向优化** | 基于 SciPy 差分进化算法的支撑最小化方向搜索 |
| **热变形模拟** | 梯度场热分析 + 规则引擎预测打印热应力 |
| **DfAM 分析** | 快速可制造性评估，输出可打印性评分与建议 |

### 2.4 平台能力

| 功能 | 说明 |
|------|------|
| **3D 在线预览** | 基于 Three.js 的浏览器内 3D 模型查看器 |
| **多格式导出** | STEP / STL / 3MF / GLB / G-code，一键下载 |
| **管线可配置** | 运行时动态调整管线节点开关、策略选择、参数配置，支持 4 种预设（快速/均衡/完整打印/自定义） |
| **历史管理** | 生成记录浏览、详情查看、版本追溯、重新生成 |
| **工业标准库** | 螺栓、齿轮、公差、键槽、法兰等标准数据，自动推荐合规参数 |
| **LLM 配置管理** | 角色化模型分配，支持运行时动态切换模型 |
| **基准测试** | 内置性能基准测试，评估生成质量与成功率 |

## 三、解决的核心问题

### 3.1 行业痛点

| 痛点 | 现状 | CADPilot 解决方案 |
|------|------|------------------|
| **CAD 建模门槛高** | 需要 3-6 个月培训才能使用 SolidWorks 等专业软件 | 自然语言输入，零门槛生成 3D 模型 |
| **设计周期长** | 一个中等复杂零件手工建模需 2-8 小时 | AI 自动生成，分钟级完成 |
| **图纸转模型低效** | 阅读图纸、理解尺寸、手动建模流程繁琐 | 图纸拍照上传，自动生成 CAD 模型 |
| **打印前处理繁琐** | 需手动检查可打印性、生成支撑、切片设置 | 全自动后处理管线：修复 → 优化 → 切片 → 出码 |
| **模型质量不可控** | 手工建模依赖个人经验，质量参差不齐 | 多层验证体系 + 自动修复 + 人机协作确认 |
| **非规则形状难建模** | 有机形状（曲面壳体、艺术造型）在传统 CAD 中耗时极长 | Neural 3D 生成通道，文本/图片直出网格 |

### 3.2 目标技术指标

| 指标 | 目标 |
|------|------|
| 尺寸精度（参数化模板路径） | < 0.5% 误差 |
| 生成成功率 | > 95% |
| 网格修复成功率 | > 95% |
| 支持零件类型 | 7 种标准件 + 有机形状 |
| 输出格式 | STEP / STL / 3MF / G-code / CadQuery 源码 |

## 七、采购必要性论证

### 7.1 为什么需要 AI 模型 API

CADPilot 的核心能力完全依赖 AI 大模型驱动，**没有模型 API 则管线无法运行**：

| 管线环节 | 依赖的 AI 能力 | 必要性 |
|---------|---------------|--------|
| **图纸分析** | qwen-vl-max 多模态视觉理解 | 必须。将工程图纸中的尺寸标注、特征描述自动提取为结构化数据，替代人工读图。这是整个图纸→模型管线的入口。 |
| **意图解析** | Qwen 系列文本理解 | 必须。将"直径50mm法兰盘"这样的自然语言转为精确的工程参数，是自然语言→模型管线的入口。 |
| **代码生成** | qwen-coder-plus 代码生成 | 必须。当参数化模板无法覆盖时，由 LLM 自动生成 CadQuery 参数化 CAD 代码，是模型生成的核心步骤。 |
| **视觉修复** | qwen-vl-max 视觉对比 | 必须。生成的 3D 模型渲染后与原始图纸对比，判断是否匹配。是质量保证的唯一裁判。 |

**选择阿里云 DashScope 的理由：**
- Qwen-VL-Max 是目前在工程图纸理解上表现最优的国产多模态模型
- DashScope 提供 OpenAI 兼容接口，代码集成成本极低
- 国内访问速度快、延迟低，适合管线中多次串行调用的场景
- 价格亲民，研发阶段月均 ¥500 即可满足需求

### 7.2 为什么需要 GPU 算力

CADPilot 的有机形态管线需要运行多个**开源 3D 生成和修复模型**，这些模型必须在 GPU 上运行：

| 需求 | 说明 |
|------|------|
| **3D 生成模型推理** | SPAR3D、TRELLIS 等开源 3D 生成模型需要 GPU 加速推理。这些模型将文本/图片转化为 3D 网格，是有机形态管线的核心。 |
| **Neural 网格修复** | NKSR 基于神经网络的曲面重建，修复质量远优于传统算法，但需要 GPU 推理。 |
| **研发调试效率** | 研发阶段需要高频次调试模型参数和管线流程，本地 GPU 服务器的响应速度和灵活性远优于 SaaS API。 |
| **长期成本优势** | 开源模型本地部署零 API 调用费用，仅需服务器租赁成本。随着使用量增长，相比 SaaS 模式可节省 60-80% 成本。 |

**2 × 24GB 显存配置的理由：**
- 单卡 24GB 可运行 SPAR3D（~12GB）或 TRELLIS（~16GB）
- 双卡可同时部署多个模型，或运行显存需求更大的 Hunyuan3D（~20GB）
- 24GB 级别的 RTX 4090 / RTX 3090 在租赁平台上性价比最高，¥2,100/月 即可满足需求
