# 白盒化工业级 AI 3D 设计中台：全链路透明化与生命周期管控方案

## 1. 核心愿景：从“黑盒生成”走向“白盒协同”

本项目旨在打造一个**过程全透明、参数全可视、逻辑可追溯、步骤可干预**的工业级 AI 设计中台。区别于传统的“一键生成”黑盒，本平台将 3D 设计的每一个微小决策点（感知、推理、编译、校验）实时暴露给用户，让工程师能够清晰定位瑕疵源头，并实现高效的人机协同纠偏。

## 2. 系统双引擎架构 (Dual-Engine Architecture)

平台在底层物理隔离两条完全独立的生成管道，分别应对不同的工业场景：

### 2.1 机械精密引擎 (Precision Engine)
*   **输入：** 2D 标注工程图纸 / 结构化参数。
*   **核心：** `VLM 感知` -> `DrawingSpec JSON` -> `SpecCompiler (确定性模板渲染)` -> `CadQuery`。
*   **目标：** 尺寸 100% 精确，逻辑可解释。
*   **降级：** 模板未命中时，自动 Fallback 至 `Coder` 模型手写代码。

### 2.2 有机概念引擎 (Organic Engine)
*   **输入：** 自然语言描述 / 概念草图。
*   **核心：** `3D Diffusion API (Tripo3D/Meshy)` -> `Mesh 后处理 (trimesh/PyMeshLab)`。
*   **目标：** 造型极致自由。
*   **工程化收口：** 利用计算几何算法在网格上“硬切削”出精确的平底和装配孔。

## 3. 白盒化 UI 设计规范 (White-box UI/UX)

为了实现“后端处理过程在前端完全透明”，UI 界面必须遵循以下三区设计：

### 3.1 管道看板区 (Pipeline Lifecycle Tree)
*   **动态节点图：** 实时展示 `[图纸分析] -> [参数化映射] -> [几何编译] -> [DfAM 检测]` 等节点。
*   **状态与耗时：** 每个节点实时显示运行状态（等待、运行中、成功、失败）及其耗时。
*   **可回溯：** 点击历史节点，中心区域立刻加载该阶段的原始输入、决策逻辑及输出结果。

### 3.2 数据与逻辑解释区 (Logic & Data Detail)
*   **意图可解释性：** 每个处理步骤配有“人话”解释（如：“检测到法兰特征，系统自动匹配了 `rotational_stepped` 建模策略，以确保轴对称性”）。
*   **参数实时表单：** 将中间状态（DrawingSpec JSON）转化为可视化表单。
    - **实时渲染：** 修改表单数值，右侧 3D 预览区通过 `SpecCompiler` 实现秒级重绘。
    - **约束提示：** 违反物理约束（如：壁厚过薄）时，表单项高亮报错并给出原因。

### 3.3 交互式预览与干预区 (Multi-View Preview)
*   **断点干预 (HITL)：** 
    - **阶段1（感知后）：** 系统强制停顿，展示 AI 读图结果，用户核对尺寸无误后点击“确认并编译”。
    - **阶段2（编译后）：** 展示生成的 Python 代码/网格预览，允许高级用户手动修改代码或调整网格定位。
*   **DfAM 高亮：** 最终 3D 模型上，通过红色热力图标识出“不可打印”或“强度脆弱”的区域。

## 4. 后端管道架构：插件化与事件流 (Pipeline Backend)

### 4.1 事件驱动的实时通信 (SSE/WebSocket)
*   后端不再返回单一响应，而是采用 **Server-Sent Events** 持续推送处理节点包。前端根据推送包实时点亮 UI 节点。

### 4.2 拦截器模式管道 (Interceptor Pipeline)
*   **插件化步骤：** 管道由一系列 `Interceptor` 组成。
*   **易扩展性：** 可以随时插入新步骤（如：`WatermarkStep` 自动打 Logo，`ThermalAnalysisStep` 热应力分析）。

---

## 5. 设计生命周期管控 (Lifecycle & Artifact Management)

### 5.1 资源全固化 (Full Artifact Traceability)
每一项设计任务生成一个唯一的 `DesignPackage`，包含：
*   **Input：** 原始图片、Prompt、草图。
*   **Intermediate：** 历次修改的 DrawingSpec JSON、生成的 Python 代码。
*   **Output：** STEP、STL、3MF、2D 渲染图、DfAM 检测 PDF 报告。

### 5.2 数据飞轮 (Data Flywheel)
*   系统自动记录用户的“纠偏动作”（如：AI 识别 10mm，用户改为 12mm）。
*   这些记录被固化为高质量的微调数据集（路径 1），用于周期性提升前端 VLM 的感知精度。

---

## 6. 后加工评价与推荐系统 (Post-Processing Guide)

当 AI 管道任务完成，系统将基于 `Printability` 结果，向用户推荐后续专业操作：
*   **机械件引导：** “检测到高精度配合面，建议导出 STEP 后在 **UG/NX** 中设置 H7/g6 公差。”
*   **有机件引导：** “模型当前为 Mesh 格式，若需 CNC 加工，请在 **Geomagic Design X** 中执行自动曲面拟合。”
*   **打印引导：** “检测到大面积悬垂，建议在 **Materialise Magics** 中生成基于热应力补偿的支撑结构。”

---

## 7. 实施路线图 (Roadmap)

1.  **Phase 1 (基础白盒化):** 实装 `SpecCompiler` 确定性路径，开发前端节点看板和参数实时修改 UI。
2.  **Phase 2 (双引擎闭环):** 完善有机曲面生成管道与 `trimesh` 强制切削逻辑，接入 Tripo3D API。
3.  **Phase 3 (生命周期与数据):** 上线数字化档案系统，建立“手动纠偏”数据收集链路，启动 VLM 微调。
4.  **Phase 4 (生态集成):** 基于工艺检测结果，上线后加工专家建议系统，并尝试对接 Oqton 等云端工业服务 API。
