# 企业级3D打印AI Agent感知模型微调实施方案

本方案旨在解决将2D工程图纸精确转换为结构化`DrawingSpec JSON`的难题。采用**“合成数据打底（路径2）+ 日常生产闭环提纯（路径1）”**的组合策略，低成本、高质量地构建微调数据集，并实施模型微调（Fine-tuning）。

---

## 阶段一：合成数据打底（路径2 - 冷启动）

**目标：** 利用程序自动化生成 10万+ 规模的 `[完美的 2D 工程图 -> 完美的 JSON]` 数据对，让模型学会“严格输出 JSON 格式”和“理解标准几何拓扑”。

### 1.1 JSON 生成器 (Data Generator)
开发一个 Python 脚本，基于 `cad3dify` 的 `DrawingSpec` 和 YAML 模板规则，随机生成合法的 JSON 参数。
- **参数随机化：** 在合法范围内（如外径 50-200mm，孔数 3-8个）进行蒙特卡洛随机采样。
- **类型覆盖：** 确保覆盖企业最常见的 7 大类零件（法兰、支架、板件等）。
- **约束校验：** 生成的 JSON 必须通过 `SpecCompiler` 的前置数学校验（如壁厚限制）。

### 1.2 3D模型批量生成
使用当前 `cad3dify` 中的确定性解析器（`TemplateEngine` + `CadQuery`），将这十万个 JSON 瞬间渲染出对应的 STEP 文件，确保每个 JSON 都有对应的物理实体。

### 1.3 2D工程图自动投影与标注
这是路径 2 的关键难点。我们需要把 3D 的 STEP 变成带标注的 2D 训练图片。
- **技术选型：** 
  - 方案A：使用 FreeCAD 的 Python API（TechDraw 模块）自动导入 STEP，生成三视图，并自动添加尺寸标注，最后导出为 PNG/PDF。
  - 方案B：利用 SolidWorks/NX 的 API（通过 COM 接口或宏脚本）进行更专业的批量自动化出图。
- **样式增强 (Data Augmentation)：** 在导出的纯净工程图上添加噪点、模拟纸张褶皱、低分辨率模糊、甚至模拟手写修改的痕迹，以增加模型在真实场景下的泛化能力。

---

## 阶段二：日常生产闭环提纯（路径1 - HITL数据飞轮）

**目标：** 将预训练好的模型放入生产环境，利用工程师日常的“修图”操作，持续收集高质量的真实脏数据，解决最后 20% 的长尾错误。

### 2.1 引入 Human-in-the-loop (HITL) 拦截界面
修改现有的 `cad3dify` 前端/后端流程，在 VLM 模型输出 JSON 后，**强制进入一个人工审核页面，不直接执行后续的代码生成。**

- **UI 设计要求：** 
  - 左侧展示客户上传的原始 2D 图纸。
  - 右侧展示 VLM 提取的结构化 JSON（以表单形式呈现，如外径：100.0，孔数：6）。
- **交互逻辑：** 工程师核对图纸与表单。如果 AI 提取错误，工程师直接修改表单数字；如果正确，直接点击“确认并生成”。

### 2.2 数据资产固化与提取功能（系统改造项）
在 `cad3dify` 项目中新增一条独立的数据沉淀链路：
1. **数据库埋点：** 在工程师点击“确认并生成”且该模型成功交付后，后台记录此次任务的生命周期数据。
2. **数据对提取：** 将 `[客户原始上传的脏图纸]` 和 `[工程师最终修改确认后的 JSON]` 自动配对打包。
3. **导出接口：** 开发一个 API（如 `/api/export-training-data`）或后台管理页面，允许算法工程师一键导出格式化好的（如 JSONL 格式）微调训练集。

---

## 阶段三：模型微调的具体实施方案 (Fine-tuning)

**目标：** 将准备好的数据集（90% 合成数据 + 10% 真实修图数据）送入开源视觉大模型进行微调，产出专属于企业的“工程感知大模型”。

### 3.1 基座模型选择
选择开源且支持多模态（Vision-Language）的小参数量基座模型，以便于在企业内部的单张或少张消费级/企业级显卡上低成本部署与训练。
- **推荐模型：** 
  - Qwen-VL-Chat (7B) 或更新的 Qwen-VL 版本（对中文和 OCR 支持极佳）。
  - LLaVA v1.5 (7B/13B)。
  - InternVL (针对高分辨率图片优化的版本)。

### 3.2 数据格式准备 (JSONL)
将阶段一和阶段二收集的数据转化为模型微调框架（如 LLaMA-Factory 或 Swift）要求的标准格式。
```json
// train_data.jsonl 示例
{
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "image", "image": "dataset/images/flange_001.png"},
        {"type": "text", "text": "请提取这张机械图纸的工程参数，严格输出为 DrawingSpec JSON 格式。"}
      ]
    },
    {
      "role": "assistant",
      "content": "{"part_type": "rotational", "overall_dimensions": {"diameter": 120.0}, ...}"
    }
  ]
}
```

### 3.3 微调策略与训练 (LoRA / QLoRA)
不建议进行全量参数微调（Full Fine-Tuning），成本过高。采用参数高效微调方法（PEFT）。
1. **训练框架：** 推荐使用 [LLaMA-Factory](https://github.com/hiyouga/LLaMA-Factory)（目前最易用的开源微调工具，支持 WebUI 界面和多种 VL 模型）。
2. **训练技术：** 使用 **QLoRA（4-bit 量化低秩微调）**。这样即使是 7B/14B 的多模态模型，也能在单张 RTX 4090 (24G显存) 或 A6000 上跑起来。
3. **超参数设置：** 
   - Learning Rate 设在 1e-4 到 2e-5 之间。
   - Epochs 建议 3-5 轮，防止过拟合到合成数据上。
   - 分辨率：由于工程图纸细节多，尽可能在训练时保持较高分辨率（如 1008x1008 或更高），防止小尺寸标注丢失。

### 3.4 评估与迭代
1. **构建测试集：** 单独保留一批未参与训练的真实图纸作为测试集。
2. **精度评估：** 评估指标不应该是常规的 BLEU 或 ROUGE（因为是结构化 JSON），而应该采用**JSON 严格解析成功率**和**关键参数误差率（Key Parameter Error Rate）**。
3. **迭代部署：** 将微调后的模型权重（LoRA weights）合并到基座模型中，替换掉 `cad3dify` V1/V2 管道中现有的在线 API 调用，在本地私有化部署。

## 4. 总结与行动项

此方案形成了一个高度闭环的技术护城河：
- **初期：** 用程序生成十万图纸，花极低的成本给模型“开蒙”。
- **中期：** 改造业务系统，通过人工审核界面保证业务质量，同时悄悄收集极高价值的真实修正数据。
- **后期：** 利用收集的数据不断微调专有模型，使其对复杂脏图纸的识别率达到工业级，彻底摆脱对外部通用大模型 API 的依赖。
