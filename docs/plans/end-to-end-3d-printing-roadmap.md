# AI 驱动的 3D 打印全流程架构设计与演进路线图

> **文档定位**: 探讨如何在 `cad3dify` 项目现有架构（AI Agent + Python 沙盒）基础上，完全依靠“模型+代码算法”替代传统工业 CAD/CAE/CAM 软件，实现从 2D 图纸/意图到 3D 打印切片文件的端到端全流程。特别是如何融合“基于工程图的参数化管线”与“基于图文提示的有机网格管线”。

---

## 1. 核心认知：视觉 3D vs 工程 3D

目前主流的生成式 3D 模型（如基于 NeRF 或 3D Gaussian Splatting 的多边形网格）主要服务于视觉展示，在工程制造中存在致命缺陷：**无精确尺寸与公差、内部可能中空或自相交、不具备参数化特征**。

在航空航天与军工领域，3D打印（增材制造）需要的是高精度的 **B-Rep（边界表示法，如 STEP）** 或适合表达复杂点阵的 **SDF（符号距离场）**。`cad3dify` 采用 AI 生成 `CadQuery` 代码的路线，天然契合工程 3D 的要求，是一条极具潜力的“圣杯”路线。

为了实现全场景支持，项目需演进为**“双大脑架构”**：
- **机械大脑 (Mechanical Brain)**: 处理工程图纸，写 Python 代码生成高精度的 STEP 模型（完美几何，绝对参数化）。
- **艺术大脑 (Art Brain)**: 处理模糊意图（草图/网图/文本），调用 3D 生成模型（如 Tripo3D）生成复杂的视觉 Mesh。这部分由于带有“脏数据”（法线错误、自相交），必须经过**强力算法清洗与水密化固化**后，才能流向后端的制造环节。

## 2. 全流程工序拆解与 Python/AI 替代方案

以下是将传统工业软件链条映射至 Python 平台内原生/AI替代方案的完整拆解。对于不同生成的源头，处理路径在“前处理”阶段截然不同，在“后处理”阶段殊途同归：

### 阶段一：概念与初步设计 (CAD Modeling)
*   **传统工具**: CATIA, NX, SolidWorks (工程) / ZBrush (有机)
*   **平台内演进方案 (机械大脑)**: 保持 `CadQuery` 路线，并引入提示词工程强制大模型输出带明确变量的参数化代码。
*   **平台内演进方案 (艺术大脑)**: 针对非标网图或文本提示，调用视觉 3D 模型 API（生成初步 OBJ/GLB），重点在于后期的混合处理（见第 3 节）。

### 阶段二：面向增材制造的设计与拓扑优化 (DfAM)
*   **传统工具**: nTopology, Altair Inspire
*   **平台内演进方案**:
    *   **机械大脑生成的模型**: 很容易。结合 `manifold3d` 依赖，利用纯数学公式生成极小曲面（TPMS），与原实心模型做布尔运算生成轻量化晶格。
    *   **艺术大脑生成的模型**: 极难直接跑拓扑优化，因为内部可能中空或乱序。必须先完成阶段三的完全水密化。

### 阶段三：模型清洗、修复与固化 (Mesh Repair - 分水岭)
*   **传统工具**: Materialise Magics, Netfabb
*   **平台内演进方案**:
    *   对于 **STEP** 转换来的模型，通常天生水密，调用 `trimesh.is_watertight` 校验后即可导出高精度 STL/3MF。
    *   对于 **AI 生成的视觉 Mesh**，这是生死攸关的一步。采用 **“包裹与体素化 (Shrink-wrapping & Voxelization)”** 算法：
        ```python
        # 降维打击，强行将残缺网格像素化再重新提取单壳体
        voxelized = mesh.voxelized(pitch=0.01).fill()
        printable_mesh = voxelized.marching_cubes
        ```
    *   结合 `pymeshlab` 自动化执行：闭合孔洞、法线统一、平滑处理。

### 阶段四：打印工艺规划与支撑设计 (Build Preparation)
*   **传统工具**: Magics SG 模块
*   **平台内演进方案**:
    *   **最佳摆放方向**: 编写 Python 寻优算法，以 $Min(Z轴打印高度) + Min(悬垂面积)$ 为目标函数遍历计算旋转矩阵。
    *   **支撑生成**: 有机模型更需要复杂的树状支撑。使用 `trimesh.ray.intersects_location`（射线检测法）从悬垂面向 Z 轴向下发射射线生成树状结构，用 `manifold3d` 合并。

### 阶段五：打印过程热力学仿真 (Process Simulation)
*   **传统工具**: Simufact Additive, ANSYS
*   **平台内演进方案**:
    *   长远来看，利用 **PINNs（物理信息神经网络）** 替代传统网格仿真，实现秒级热变形预测。
    *   短期内利用 Python 脚本封装开源 FEA 引擎（如 `CalculiX`）在后台运行，反向补偿原始 Mesh 节点。

### 阶段六：切片与路径规划 (Slicing & Toolpath)
*   **传统工具**: EOSPRINT, CuraEngine
*   **平台内演进方案**:
    *   **自研算法**: 利用 `trimesh.intersections.mesh_plane` 在特定 Z 高度截取 2D 轮廓。引入几何库 `Shapely` 生成外壳偏置线和内部填充（工程件高密度走线，有机件闪电填充）。
    *   **集成开源引擎**: 利用 `subprocess` 在后台直接调用 CuraEngine 的 CLI 工具，实现静默切片。

## 3. 高阶商业刚需：非标视觉输入 -> 工程级输出 (混合几何管线)

用户常有此类需求：“用网上一张炫酷的头骨模型图（艺术外观），做成汽车排挡杆，底部必须带有直径 12mm 深度 30mm 的精确安装孔（工程装配）”。
这要求系统跨越单纯的翻译，融合感性（网图）与理性（硬参数）。

**破局方案：布尔拼装法 (Hybrid Geometry Pipeline)**

这是目前 AI for 3D 最好落地的方案：
1. **艺术大脑构建造型**: 调用视觉 API，将概念图生成基础有机 Mesh。
2. **清洗与尺寸约束 (Python 算法)**: 使用体素化算法将有机 Mesh 强行变成无相交的单壳水密实体，并强制缩放其包围盒（Bounding Box）到合理的物理尺寸。
3. **机械大脑构建接口**: 提取装配参数，通过大模型生成 `CadQuery` 代码，生成一个精确的“虚拟切削刀具”（例如绝对精确的 M12 螺纹底孔模型）。
4. **融合运算**: 利用 `manifold3d` 极速进行**布尔相减 (Boolean Difference)**，将水密有机实体减去精准工程特征，获得兼具艺术外形与工业装配公差的可打印综合模型。

## 4. 针对 `cad3dify` 架构的近期行动建议

1. **新增 Stage 3.8 (DfAM & Print Ready 循环)**：
   在管道（`Stage 3.5: 几何验证` 和 `Stage 4: 智能改进` 之间），加入算法物理约束：

   在对接直接基于图文提示生成的路线中，正式引入基于 `trimesh.voxelized` 和 `pymeshlab` 的强固化脚本流，作为通向打印后处理的必经“安检站”。

## 5. 系统架构设计建议：模块化与可插拔管线

考虑到在实际工业应用中，用户往往不需要走完所有 6 个步骤（例如，仅需完成阶段一导出 STEP 文件后，交由专业软件继续处理），系统架构必须具备高度的**灵活性、可插拔性**以及**状态可追溯性**。

### 5.1 核心设计理念

1.  **节点化 (Node-based) / 阶段化执行**：将前述的 6 大步骤抽象为独立的 Pipeline 节点。每个节点都有明确的输入和输出契约。
2.  **提前熔断与导出 (Early Exit & Export)**：允许在任意节点成功后，直接输出该阶段的中间产物（如 STEP, 原始 OBJ, 水密 STL）。
3.  **状态机与溯源 (State Machine & Lineage)**：每一次生成任务都应是一个状态机，记录当前经历的节点、消耗的时间、以及生成的中间资产，形成完整的数据血缘。

### 5.2 架构模式建议：有向无环图 (DAG) 驱动的 Pipeline

建议放弃线性代码调用，改用基于 DAG（如引入更轻量的状态机框架如 `langgraph` 或自研简易基于步骤的编排器）。

**核心数据结构示例：**

```python
class PipelineState:
    job_id: str
    target_stages: list[str]  # e.g., ["modeling", "repair"]
    current_stage: str
    assets: dict[str, str]    # 记录中间产物: {"step_model": "path.step", "repaired_stl": "path.stl"}
    lineage: list[dict]       # 记录执行历史: [{"stage": "modeling", "status": "success", "duration": 45.2}]
```

### 5.3 典型的用户场景与管线配置

通过暴露给用户的配置，系统可以动态组装需要执行的步骤：

*   **场景 A：传统 CAD 辅助 (只要模型，专业软件后处理)**
    *   `target_stages = ["concept_modeling"]`
    *   **执行**：步骤 1 -> 结束。
    *   **输出**：`.step` 文件。
*   **场景 B：直接 3D 打印 (全托管傻瓜模式)**
    *   `target_stages = ["concept_modeling", "mesh_repair", "build_prep", "slicing"]`
    *   **执行**：步骤 1 -> 步骤 3 -> 步骤 4 -> 步骤 6 -> 结束。
    *   **输出**：`.gcode` 切片文件（可以直接喂给打印机）。
*   **场景 C：非标网图混合建模 (需要艺术大脑+修复)**
    *   `target_stages = ["organic_generation", "mesh_repair", "boolean_fusion"]`
    *   **执行**：艺术大脑生成 -> 步骤 3 (体素化水密) -> 步骤 1 (机械接口) -> 布尔运算 -> 结束。
    *   **输出**：可打印的混合 `.stl` 文件。

### 5.4 状态追溯设计

由于步骤可选且随时可能因失败而导出中间态，必须有一个健壮的持久化机制（目前项目中的 `sqlalchemy` 和 `aiosqlite` 非常适合做这件事）：

1.  **Job 表**：记录总体状态、用户意图、选定的 `target_stages`。
2.  **Job_Stage_Log 表**：记录 `job_id`，当前阶段，输入物路径，输出物路径，失败日志。这不仅对用户展示进度（例如进度条、节点高亮）至关重要，也为未来微调模型收集了极其宝贵的数据对（例如：哪些图纸在阶段一容易失败，哪些模型在阶段三修复时消耗最多算力）。

## 6. 理性成功率分析与核心价值杠杆

针对中低复杂度模型（Low-to-Medium Complexity），基于上述“双大脑 + 模块化”架构，我们对系统的真实可用性进行量化评估。成功分为两级：**Tier 1（完美成功，直接上机打印）**与 **Tier 2（辅助成功，需工程师微调<10分钟）**。

### 6.1 管线一：机械大脑（2D工程图 -> 代码 -> STEP）
目标群体：法兰、轴、简单支架、外壳底座等特征数为1-6的机械件。

*   **直接打印成功率 (Tier 1)：约 30% - 40%**
    *   瓶颈不在几何内核，全在 VL 模型的“读图视力”。当前视觉大模型对工业图纸的数字、公差、形位公差及多视图关联存在约 5%-10% 的“幻觉率”。一旦读错一个关键尺寸，直接打印即为废品。
    *   CadQuery 生成的 STEP 是完美的（100%水密），只要形状正确即可完美切片。
*   **微调后可用率 (Tier 2)：约 80% - 90% （核心商业价值）**
    *   系统输出的并非死模型，而是结构正确的、参数化的 CadQuery 代码和特征树。
    *   工程师只需修改代码中提取错的变量（如把 `hole_diameter = 1.2` 修正为 `12`），重新运行只需几秒钟。这比从零画草图效率提升 5 倍以上。

### 6.2 管线二：艺术大脑（图+文提示 -> Mesh -> 水密固化）
目标群体：手办、概念外壳、特定造型把手等非标/有机件。

*   **直接打印成功率 (Tier 1)：约 60% - 70%** (针对 FDM/树脂等非严苛公差打印)
    *   由于管线强制加入了“体素化水密清洗”节点，彻底解决了视觉模型最致命的网格破损问题。切片软件不会报“不闭合”错误。
    *   失败主因在于物理结构不合理（如悬垂过长过薄），算法难以为违背重力的结构自动生成完美修复。
*   **混合几何（布尔拼装）直接成功率：约 40% - 50%**
    *   机械大脑的孔与艺术大脑的造型融合时，纯靠算法“对齐”可能导致打偏或切掉重要特征。
*   **混合几何微调可用率 (Tier 2)：>90%**
    *   系统提供“水密艺术件 STL”和“精准机械孔 STEP”，工程师在 Magics 或 NX 中手动调整对齐并进行布尔减运算，1分钟即可完成，大幅降低逆向工程成本。

### 6.3 系统的核心发力点（护城河）
1.  **对于管线一：强化参数化提取与交互。** 放弃死磕 100% 的 AI 视觉识别率，转而在 Web UI 提供直观的变量编辑表格。让用户一键修改 AI 提取的特征变量并极速重渲。
2.  **对于管线二：死磕几何固化算法。** 深度优化 `trimesh` 和 `manifold3d` 的清洗工作流。保证平台流出的每一个模型绝对水密、无拓扑错误，在“可打印性”上对玩具级 Text-to-3D 平台形成降维打击。
