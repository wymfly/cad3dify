# CAD3Dify V2 当前版本能力说明

> **版本:** V2 (2026-02-26 基线)
> **管道:** VL 读图 → 策略选择 → Coder 写码 → 几何验证 → 智能改进
> **定位:** 将 2D 工程图纸自动转换为 3D STEP 模型

---

## 1. 管道架构

```
输入: 2D 工程图纸 (PNG/JPG)
  │
  ├─ Stage 1: DrawingAnalyzer (qwen-vl-max, temp=0.1)
  │    └─ CoT 推理 → DrawingSpec (零件类型 + 尺寸 + 特征)
  │
  ├─ Stage 1.5: ModelingStrategist
  │    └─ 策略指南 + Jaccard 特征匹配选择 few-shot 示例
  │
  ├─ Stage 2: CodeGenerator (qwen-coder-plus, temp=0.3)
  │    └─ Spec + 策略 + 示例 → CadQuery Python 代码
  │
  ├─ Stage 3: 代码执行
  │    └─ 沙箱执行 CadQuery 代码 → STEP 文件
  │
  ├─ Stage 3.5: 几何验证
  │    └─ STEP 有效性 + 体积 + 包围盒
  │
  └─ Stage 4: SmartRefiner (最多 3 轮)
       ├─ Layer 1: 静态参数校验（诊断）
       ├─ Layer 2: 包围盒校验（诊断）
       └─ Layer 3: VL 对比（唯一裁判）→ Coder 修复
```

**关键设计决策:**
- SmartRefiner 采用零风险模式：Layer 1/2 仅提供诊断上下文，VL 始终运行
- VL 是质量的唯一裁判，静态检查不会跳过或替代 VL 判定
- V1 管道保留作为 fallback（Stage 1 解析失败时自动降级）

## 6. 能力边界：能做什么

### 擅长处理的图纸

1. **简单回转体**：法兰盘、轴、轴套 — 明确的阶梯轮廓 + 标准特征（孔、圆角）
2. **矩形板件**：安装板、底板 — 规则几何 + 孔阵列
3. **标准支架**：L/U/T 形 — 正交拉伸组合
4. **标注清晰的图纸**：尺寸标注完整、视图规范（正视图/剖视图/俯视图）

### 中等把握的图纸

1. **薄壁壳体**：矩形/圆柱壳体 — 需要正确的抽壳操作序列
2. **简化齿轮**：矩形齿近似 — 不追求渐开线精度的场景
3. **带多特征的零件**：同时包含孔阵列+圆角+倒角的中等复杂度零件
4. **非标准视图布局**：视图排列不完全规范但尺寸标注完整的图纸

### 复杂度评估

| 复杂度 | 特征数 | 预期成功率 | 典型零件 |
|--------|-------|----------|---------|
| 简单 | 1-3 个特征 | ≥80% | 单台阶轴、安装板、管件 |
| 中等 | 4-6 个特征 | 50-70% | 阶梯法兰盘、带筋底板、L支架 |
| 复杂 | 7+ 个特征 | <40% | 多孔箱体、腹板齿轮、复杂壳体 |

> 注：成功率为估算值，尚未建立正式评测基准。

## 7. 已知局限

### 7.1 不支持的几何特征

| 特征 | 原因 | 影响 |
|------|------|------|
| 渐开线齿形 | 齿轮示例使用矩形近似 | 齿轮精度不足，仅适合展示 |
| sweep/loft（扫掠/放样） | 无示例覆盖 | 无法处理变截面零件 |
| 自由曲面 | CadQuery 限制 + 无示例 | 无法处理造型类零件 |
| 钣金折弯 | 无示例覆盖 | 无法处理钣金件 |
| 螺纹 | 无示例覆盖 | 只能用简单孔代替 |
| 装配体 | 管道仅处理单零件 | 多零件图纸无法处理 |
| 弹簧/弹性体 | 无示例覆盖 | 无法处理 |

### 7.2 图纸识别局限

| 局限 | 说明 |
|------|------|
| 手绘图纸 | VL 模型对手绘线条识别率低 |
| 低分辨率/模糊图纸 | 尺寸标注可能误读 |
| 非标注视图关联 | 多视图间的对应关系依赖 VL 推理能力 |
| 公差/表面粗糙度 | 可识别但不影响 3D 建模 |
| 第三角投影 vs 第一角投影 | 未显式区分 |
| 局部视图/放大图 | 未显式支持 |

### 7.3 管道弱点分析

| 阶段 | 弱点 | 影响程度 |
|------|------|---------|
| Stage 1 (VL 分析) | 纯视觉推理，无 OCR 辅助；单次分析无自一致性校验 | 高 — 尺寸误读直接导致后续全错 |
| Stage 1.5 (策略选择) | Jaccard 匹配粒度粗，知识库仅 20 个示例 | 中 — 复杂零件可能匹配到不合适的示例 |
| Stage 2 (代码生成) | 单次生成无 best-of-N；无 API 白名单约束 | 中 — 可能使用不存在的 API |
| Stage 3.5 (几何验证) | 仅验证有效性和包围盒，无体积/拓扑检查 | 低 — 检查粒度不够细 |
| Stage 4 (SmartRefiner) | 单视角渲染对比；无回退机制 | 中 — 遮挡角度可能漏检问题 |

## 8. 与专业 CAD 软件的差距分析

> 对标 Siemens NX / SolidWorks 等专业软件，从 3D 打印输出视角评估。

### 8.1 核心差距：设计意图传递

差距的本质**不在 CAD 内核**。CadQuery 底层使用 OpenCascade（与 NX 的 Parasolid 同级别的工业内核），几何建模能力本身足够。真正的瓶颈在于「设计意图传递链」中的两层有损压缩：

```
专业软件:  设计师大脑 → 手动操作 → 精确参数化特征树
cad3dify:  图纸像素 → VL 猜测（有损）→ 自然语言 Spec → LLM 生成代码（概率性）→ 特征树
```

- **VL 读图是有损的** — 精确的工程标注被降级为 VL 的「理解」，当前误差 5-10%
- **LLM 代码生成是概率性的** — 同一个 Spec 每次生成的代码可能不同

### 8.2 几何精度对比

| 维度 | 专业 CAD 软件 | cad3dify V2 |
|------|-------------|-------------|
| 尺寸精度 | 精确到 μm 级 | 依赖 VL 读图，误差 5-10% |
| 曲面质量 | NURBS 精确表达，G2 连续 | OCCT 内核能力足够，但 prompt 驱动无法精确控制曲面参数 |
| 齿形 | 渐开线精确参数化 | 矩形近似，打印后无法啮合 |
| 公差 | 完整 GD&T 体系 | 完全忽略 |
| 参数化关联 | 全约束参数化，修改联动 | 一次性代码生成，无参数关联 |

### 8.3 3D 打印就绪度对比

| 3D 打印关注点 | 专业 CAD 软件 | cad3dify V2 |
|-------------|-------------|-------------|
| 壁厚均匀性 | 设计师精确控制 | 抽壳操作可能不均匀，无检查 |
| 悬挑角度 | DfAM 模块分析优化 | 完全不考虑打印可行性 |
| 支撑结构需求 | 可分析并最小化 | 无感知 |
| 最小特征尺寸 | 可按打印机精度约束 | 不感知打印机参数 |
| 输出格式 | 原生 STL/3MF，可控网格密度 | 仅输出 STEP，需额外转换 |
| 多零件装配 | 完整装配建模 | 仅支持单零件 |
| 内部结构（晶格/填充） | 高级 DfAM 支持 | 无能力 |

### 8.4 适用场景定位

**适合的场景：**
- 有图纸、需要快速得到「大致正确」的 3D 模型用于预览或讨论
- 简单零件的快速原型（法兰盘、轴、安装板），打印后用于验证外形
- 批量处理旧图纸的 3D 化（不要求高精度）

**不适合的场景：**
- 需要精确装配的功能性零件
- 需要啮合的齿轮/传动件
- 对尺寸精度有明确要求的场景（当前 5-10% 误差）
- 需要 DfAM 优化的 3D 打印专用设计

### 8.5 缩小差距的优先级

| 优先级 | 方向 | 目标 | 对应质量优化方案 |
|-------|------|------|---------------|
| P0 | 尺寸精度 | 误差从 5-10% 降到 1-2% | 1.1 OCR 辅助 + 1.5 Self-consistency |
| P1 | 输出格式 | 增加 STEP → STL/3MF 转换 | 新增（当前路线图未覆盖） |
| P2 | 可打印性检查 | 壁厚、悬挑角度、最小特征验证 | 新增（当前路线图未覆盖） |
| P3 | 渐开线齿轮 | 参数化公式替代矩形近似 | 1.5.1 知识库扩充 |
| P4 | 参数化输出 | 生成可二次编辑的参数化代码 | 1.5.3 参数化模板 |

## 11. 总结

**当前 V2 管道能够处理尺寸标注清晰的简单到中等复杂度回转体和板件类零件，对标准特征（阶梯、通孔、孔阵列、圆角/倒角）有较好支持，但对复杂几何（渐开线齿轮、扫掠/放样、自由曲面）和非标准图纸（手绘、模糊、局部视图）能力有限。与专业 CAD 软件的核心差距在于设计意图传递链的有损压缩（VL 读图误差 5-10%）和 3D 打印就绪度不足（无 DfAM、仅输出 STEP）。质量瓶颈在 VL 读图精度和知识库覆盖范围，而非管道架构或 CAD 内核本身。**
